We have particulary two versionning control such as git and mercurial

git innit: to innitialise the project
git status : to see the status of the changes after mek changes
git add file1.txt file2.txt or git add *: to add file in the local area
git restore --staged . : to cancel all the last commit after add it
git clean : undo local changes
git status -s: to see the status output such as (added:A) (modified:M) (deleted:D)
git commit -m: to commit your changes(or change it)
git  commit -am "Fix the bug to signUp connexion"

git rm file2.txt :to remove file in the directory
git restore --source=HEAD~1 file.txt : to restore the file to the previous version
git mv main.js index.txt :rename the file to the respository
   
 .gitignore (git add gitignore) :This file is use to add files and directories that will be ignore 
 	eg:echo logs/ > .gitignore
 
  git ls-files : show files in the status area
  ---look for the gitignore according to your project to known types of files that are suppose to been removed or exclude when push your code
  
diff --git a/file.js b/file.js: give the difference betweeen 2 files

git log:history commit start from earliest to the lastest
